ปัญหาในการใช้

25-11-23 — นายเพนกวิน

เนื่องจากผมเขียน bashblog นี้ด้วย แลปทอปวินโดวน์ ซึ่งไฟล์ bash นั้นทำงานในลินุกซ์ ซึ่งผมก็มีแลปทอปอีกตัวแต่มันมันจะนอนอยู่ห้อง ผมก็ไม่อยากหยิบมา

จึงต้องใช้ wsl ในการทำ โดยขั้นแรกต้องหาไดเรกทอรี home ก่อน ซึ่งจะอยู่ที่ \wsl$\Ubuntu\home\ชื่อผู้ใช้ และจากนั้นให้ไปพาธของ bb.sh ของผมอยู่ในไดรฟ C: ก็จะเป็น /mnt/c จากนั้นให้ลองรัน bb.sh หากไม่ได้ ก็กลับไปใช้แลปทอปที่ห้อง

หยอก ๆ 5555

จากนั้นถ้าลองรัน bb.sh แล้วแต่ไม่ได้ ให้แปลงข้อความใน bb.sh ด้วย sed -i 's/\r$//' bb.sh ผมลองแล้ว มันโอเค

ส่วนตอนนี้ post หาก markdown ไม่ทำงาน ให้แปลงข้อความในไฟล์ Markdowm.pl ด้วย sed -i 's/\r$//' Markdown.pl

เท่านี้ก็เป็นอันเสร็จ

ก่อนหน้านี้ผมใช้ Nano และ Git Bash ผ่าน Terminal ใน VSCodium แต่มันมีปัญหาเรื่องอักษรภาษาไทย อักษรมักจะมีการบรรทัดเพี้ยน

ตอนนี้ใน wsl ผมใช้ vim แบบไม่มี config พิเศษอะไรเลย

เพราะว่ามันยุ่งยาก ผมจึงใช้มัน ชีวิตจะได้มีสีสัน ไม่น่าเบื่อ

การโพสต์ย้อนหลัง

25-11-23 — นายเพนกวิน

เนื่องจากปกติแล้ว ผมเขียนบันทึกเรื่องราว ความคิด ความเห็น ด้วยคอมพิวเตอร์อยู่บ้าง โดยเขียนในรูปแบบของไฟล์ .txt

ตอนนี้ผมต้องการที่จะย้ายไฟล์เหล่านั้นมาอยู่ในบล็อกของผม วิธีการคือ... การสร้างไฟล์ .md ในโฟลเดอร์ร่าง

จากนั้นใช้คำสั่ง ./bb.sh post แล้วจะได้ไฟล์ .html มา ต่อจากนั้น ให้เป็นแก้ timestamp ที่คอมเมนต์อยู่ในไฟล์ html โดยเลือกเปลี่ยนปีเดือนวันเอา จากนั้นก็ ./bb.sh rebuild ก็เป็นอันเสร็จเรียบร้อย

การติดตั้ง Markdown ใน bash blog

25-11-09 — นายเพนกวิน

ตอนแรกผมก็งง ว่าเวลา รันคำสั่งสำหรับการสร้างบล็อกทำไมไฟล์แก้ไขไม่เป็น .md แต่เป็น .html ซึ่งมันก็จะลำบากนิดหน่อยในการพิมพ์แท็ก (จริง ๆ ก็ไม่อยากมาก แต่ .md มันสะดวกกว่ามาก เพียงแค่พิมพ์ไปตรง ๆ เลย ถ้าไม่เน้นข้อความอะไร) พอไปอ่านไฟล์ bb.sh เห็นว่าต้องมี Markdowm.pl เราก็เลยไปค้นมาในเน็ต mackyle/markdown โหลดมาแล้วก็ใส่ในไดเรกทอรี จากนั้นตูม!! เรียบร้อย
