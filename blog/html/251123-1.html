ปัญหาในการใช้

Date: 2025-11-23 - นายเพนกวิน

เนื่องจากผมเขียน bashblog นี้ด้วย แลปทอปวินโดวน์ ซึ่งไฟล์ bash นั้นทำงานในลินุกซ์ ซึ่งผมก็มีแลปทอปอีกตัวแต่มันมันจะนอนอยู่ห้อง ผมก็ไม่อยากหยิบมา

จึงต้องใช้ wsl ในการทำ โดยขั้นแรกต้องหาไดเรกทอรี home ก่อน ซึ่งจะอยู่ที่ \wsl$\Ubuntu\home\ชื่อผู้ใช้ และจากนั้นให้ไปพาธของ bb.sh ของผมอยู่ในไดรฟ C: ก็จะเป็น /mnt/c จากนั้นให้ลองรัน bb.sh หากไม่ได้ ก็กลับไปใช้แลปทอปที่ห้อง

หยอก ๆ 5555

จากนั้นถ้าลองรัน bb.sh แล้วแต่ไม่ได้ ให้แปลงข้อความใน bb.sh ด้วย sed -i 's/\r$//' bb.sh ผมลองแล้ว มันโอเค

ส่วนตอนนี้ post หาก markdown ไม่ทำงาน ให้แปลงข้อความในไฟล์ Markdowm.pl ด้วย sed -i 's/\r$//' Markdown.pl

เท่านี้ก็เป็นอันเสร็จ

ก่อนหน้านี้ผมใช้ Nano และ Git Bash ผ่าน Terminal ใน VSCodium แต่มันมีปัญหาเรื่องอักษรภาษาไทย อักษรมักจะมีการบรรทัดเพี้ยน

ตอนนี้ใน wsl ผมใช้ vim แบบไม่มี config พิเศษอะไรเลย

เพราะว่ามันยุ่งยาก ผมจึงใช้มัน ชีวิตจะได้มีสีสัน ไม่น่าเบื่อ
